Syntax: JS

Basic Syntax

To use Simplistic Design, there is a basic syntax that you must follow. Of course, the normal HTML syntax applies, so you have to follow that. The basic order that the main elements in the page should go in are as follows:
body, dialogs, nav, header, wrapper, (inside wraper): article, aside, (outisde wrapper): footer.

Of course, all of these elements are optional, and you can technically put the elements in any order that you want, you just might get some funky results. What I put above this is simply a recommendation. The only thing that is 100% required is that you have a <div class="wrapper">...</div> that surrounds the article and aside elements.

What if I have an Aside?

If you wish to put an aside, you must add the attribute hasAside="true" to your wrapper. This will tell the code to shrink the main article and to put the aside on the actual side of the page. When the page is in mobile, however, the article will appear in full screen so as to make a more pleasant reading experience for those on mobile. The aside, then, will appear at the bottom of the screen. If you fail to add the attribute hasAside="true" to the div, then the aside will appear at the bottom of the screen no matter what platform the user is viewing the page on.
