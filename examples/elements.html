Obviously, the nav element makes the bar in the first place. The class show-header, as mentioned in the text above the code, allows the navigation bar to go transparent when the body is at the top, so that the user can see the header. Adding the attribute disabled to the button makes sure that the user can't press the button of the page that they are currently viewing, as well as indicating that by underlining the button. The data-icon attribute specifies the icon that should display when the site is viewed in mobile (this will be displayed when the user expands the nav bar). Simplistic Design only requires that you put the icon, as it automatically adds "fa fa-" before the icon name that you specify. These icons come from Font Awesome. NOTE: If you do not have the data-icon property, the page will not work in mobile. You can verify this by pressing Ctrl + Shift + I or F12, showing the developer's console. If you go to the "Console" tab, you will be able to see any errors sent by Simplistic Design, Beginning with SD Error:.

Ripple Effect

The ripple effect is a simple animation that gives the user of a page feedback that they have clicked the button and that the page has registered that click. It is very useful and gives a nice look and a fun feel to your web site.

It also works on FABs!
